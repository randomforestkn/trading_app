# Screenshot Capture Guide

Capture screenshots in dark mode only. Keep the demo data polished and avoid
visible debug or placeholder text.

## Recommended device sizes

- Android phone: 1080x2400 or similar tall phone size
- iPhone: 1170x2532 or similar tall phone size
- Tablet: 1280x800 or wider
- Desktop / macOS: wide window with clear spacing

## Screens to capture

- Home
- Portfolio
- Analytics
- Strategy simulator
- Options portfolio
- Journal
- Insights
- Export / Restore
- Settings / Diagnostics

## Demo data preparation

- Reset the paper portfolio if you want a clean starting point.
- Keep a few open positions visible.
- Ensure market refresh has been run so charts and changes are populated.
- Add a small number of journal entries and option positions for context.
- Keep the options and insights warnings readable but not overwhelming.

## Clean-state recommendations

- Start from onboarding accepted mode so the main shell opens immediately.
- Avoid empty states unless the screenshot is specifically for an empty-state
  gallery.
- Clear noisy history if the screen should emphasize layout and not dense data.

## Styling guidance

- Use the app in dark mode.
- Keep headers, cards, and stat tiles aligned.
- Avoid showing keyboards in screenshots unless a form is the point of the shot.
